00 Cross-cutting 9 issues

Patterns that show up across every module and are best fixed once, centrally.

P0

Type scale is far too small for a daily-use workspace

Body text sits at 10–12px across almost every screen (Sidebar labels 11px, header title 13px, table cells 11px, form labels 9–10px at letter-spacing 1.5–2.5px). For staff using this 6+ hours a day — often on a packing counter under fluorescent light — this is fatiguing and fails WCAG 1.4.4 readability heuristics.

Affects: every module lib/theme.tsx Fix: bump base to 13px, labels to 11px, tx3 → tx2 for secondary text
P0

Destructive actions use window.confirm() with no undo

Delete SKU, delete scan from history, void challan, and delete expense all fire a native browser prompt then commit. Given operator-role users work fast, a mis-tap on row "×" means irrecoverable loss. The app already has a toast system — use it for 5-second undo on delete instead.

BrandTagPrinter.tsx:deleteRow CashChallan.tsx:void/delete CashBook.tsx:pendingExpDel Fix: optimistic delete + toast with Undo, 5s window (already implemented for expenses — generalize)
P1

Inline styles everywhere — no single place to change anything

Every component has hundreds of lines of style={{...}}. Theme tokens help but the pattern still means spacing, radius, focus rings, button heights, and hover states are repeated 20+ times per file. Changing "button radius from 6 to 8" today requires touching every component.

Fix: lift recurring recipes (btnPrimary, btnGhost, inp, thS, tdS, Card, Modal) into theme.tsx's S object and use everywhere
P1

Mobile bottom nav hides Brand Tags and Settings

App.tsx:104 only surfaces Dashboard / Inventory / PackStation / Challan on mobile. Admins on phones cannot reach Brand Tags or Settings at all without desktop. There's no hamburger either — the drawer exists but has no trigger visible in the mobile layout.

App.tsx Fix: add a 5th "More" tab opening the full drawer, or show hamburger in header on <900px
P1

No focus-visible ring on any interactive element

Buttons, rows, and icon triggers are all <div onClick> or <button> with inline style and no :focus-visible. Keyboard users are locked out of sidebar navigation, tab switching, row actions, and modal buttons. Also hurts screen-reader users because divs with onClick aren't announced.

Fix: global :focus-visible{outline:2px solid var(--ac);outline-offset:2px} + swap nav divs to <button> with role semantics
P1

Glass-morphism is applied inconsistently and hurts text contrast

Header, sidebar, and card backgrounds use rgba(8,11,20,.6) + blur(32px). On the ambient-glow background the blur creates a muddy surface where --tx3: #4A5568 labels drop below 3.5:1 contrast. Worst on login card placeholders and sidebar "Menu" eyebrow.

Fix: keep glass only on header+sidebar; cards go to solid --s2. Bump --tx3 to #6B7890 (4.6:1).
P2

Empty states are all text, never illustrated or actionable

"No rows. Import Excel or add SKUs." / "No records found." / "No notifications" — all centered grey text. A user landing on a fresh Brand Tags page has no visual cue where to start; the "+ Add SKU" button is in the top-right corner.

Fix: empty-state card with icon, one-line instruction, and primary CTA — reusable <Empty> component
P2

Error messages leak backend details

alert('Save failed: ' + error.message) and alert('Update failed: ' + error.message) surface raw Postgres/Supabase strings like "duplicate key value violates unique constraint …". Also: alert() breaks the app's visual language.

Fix: map known error codes to human copy ("This SKU is already registered"), send raw to console only, surface via toast
P3

Ambient background glows re-render on every route

Four blurred orbs + dot grid are painted in App.tsx root on every tab. Cheap but still pushes layout paint on tab change. Move to a fixed-position layer outside the lazy-mount pages.

01 Dashboard 5 issues

pages/Dashboard.tsx — Today's Pulse, alerts, revenue trend, top customers, tasks.

P1

"Today's Pulse" card layout buries the one number that matters

Scans, Revenue, Cash Available, and Unsorted all render at the same size and colour. The owner's question is always "how much cash came in today?" — that number should be 3× bigger and anchor the screen. Right now it's a 26px figure in a 1-of-4 grid cell.

Fix: hero-ify Revenue; demote Scans/Unsorted to secondary stat strip
P1

Alerts and Top-5 customers are not clickable

The dashboard shows "3 overdue invoices, ₹12,400" and "Top customer: Rao — ₹8,400 outstanding" but clicking them does nothing. Every dashboard number should deep-link to the filtered list view (Cash Challan filtered to unpaid, or Inventory filtered to dry_clean > 7 days).

Fix: each alert row = button that sets tab + pre-applies filter
P2

Revenue trend chart lacks y-axis reference

30-day sparkline has no baseline, no peak label, no weekend shading. A spike from a single big sale looks identical to a week-long trend. Add min/max dots with ₹ labels and a faint 7-day moving-avg line.

P2

Tasks section has no "Add task" from Dashboard

Shows open tasks but to create one you leave the dashboard. Small inline input at the bottom of the list would let users capture follow-ups without context-switching.

P3

Date label "Today · 4 Dec" is small and grey

For a screen users glance at every morning, the date should feel present — bump to 16px, white, and add day-of-week.

02 Inventory 6 issues

pages/Inventory.tsx + InventoryExtras.tsx — item list, status pipeline, batch ops.

P0

Status transitions have no confirmation or audit-visible diff

Changing an item from "sorted" to "dry_clean" to "damaged" writes to status_changed_at but the UI offers no "undo last change" and no visible history on the row. Given damage/dry-clean statuses trigger Dashboard alerts, an accidental status flip skews the operations view silently.

Fix: inline status-history popover on each row showing last 5 transitions + who/when
P1

Filter bar steals 3 lines of vertical space without being collapsible

Search + category + location + status + severity + date range all sit permanently above the table. On a 13" laptop only 4–5 rows are visible at once. Collapse secondary filters behind a "Filters" popover with a count badge.

P1

Bulk actions are not discoverable

Rows have checkboxes but no visible "batch move / batch status change" control appears until you've selected one. And even then it's a small button row above the table — easy to miss. Docked action bar at the bottom of the viewport (like Gmail) is the established pattern.

P1

Barcode scan hand-off is inconsistent with PackStation

Header scan icon opens a scanner, on hit routes to Inventory and sets notifItemId → auto-opens the detail modal. PackStation has its own camera flow. The two use different barcode libraries (BarcodeScanner.tsx vs barcode-detector/ponyfill) and different beeps. Pick one.

P2

Status pills blend into each other

unsorted / sorted / dry_clean / damaged / packed / shipped — six pills, all at opacity ~.1, all 10px. At a glance the column looks like noise. Use a tiny coloured status-dot + plain label for scannability; reserve pills for modal emphasis.

P3

Column widths are not user-adjustable or persistent

Long SKUs and addresses truncate with no tooltip. Either allow drag-to-resize with localStorage, or add title attributes on truncated cells.

03 Brand Tags 5 issues

BrandTagPrinter.tsx — SKU master, label printing, order-sheet matcher.

P0

Label print window uses window.open + alert on popup block

If the browser blocks the popup, the flow dies with a raw alert and the user has no second path. On a staff laptop with strict policies this is a real blocker for printing labels.

BrandTagPrinter.tsx:printLabelsInWindowFix: render labels in an in-app modal iframe, call window.print() on the iframe — no popup needed
P1

Order-sheet import blocks "Print All" if any row is missing

Correct intent, but the block is all-or-nothing — user can't "print 47 of 48, resolve 1 later". The UI already knows which rows are ready. Allow printing the ready subset with a loud confirmation of how many are held back.

P1

Toolbar wraps onto 2 rows on 13" laptops

Refresh / Import / Export / Order Sheet / Test Print / + Add SKU — 6 buttons on the right plus the title and count on the left. At 1280px the whole bar breaks. Group secondary (Import/Export/Test/Refresh) into a "More" menu.

P2

Import validation error dialog truncates at 10 rows

"...and 847 more" is demoralising and not actionable. Offer a "Download error report (.xlsx with reasons column)" button in the alert so the user can fix the file in Excel.

P2

Modal has 10 fields in a single column — no grouping

Brand / EAN / SKU / Product / Includes / Size / Color / MRP / MKTD / Jio Code — all at equal weight. Group into "Identity (Brand, SKU, EAN, Jio)", "Description (Product, Includes, Size, Color)", "Commercial (MRP, MKTD)". Two columns on wide screens.

04 PackStation 6 issues

PackTime.tsx — high-frequency scanning flow, 100s of scans/day.

P0

Duplicate-scan modal blocks for 1.5s with no skip

On a duplicate the whole screen dims, a big ⚠️ modal auto-dismisses after 1500ms. During a burst of scans this wrecks throughput and forces the operator to wait. Duplicate feedback should be a non-blocking red flash on the AWB strip + error beep — modal reserved for a hard stop.

PackTime.tsx:duplicateAwb
P1

Setup screen asks three questions but only shows one result

Brand + Courier + Camera. Once "Start Scanning" is hit there's a Verify spinner that could take seconds. If the sheet column order is wrong the user gets a modal-style error but has no "Retry" or "Fix headers for me" action.

P1

"Change" setup button disabled mid-session with only a tooltip

sessionCount > 0 disables Change with a title attribute. On mobile tooltip is invisible. If the operator set the wrong courier on scan #1 they're trapped. Replace with a confirm-dialog that warns and continues.

P1

Recent-scans strip shows only 30 items and no session summary

After 100 scans the operator has no way to skim what's been done this session without opening History. Add a "Session list" collapsible with full scan list + CSV export.

P2

Pending-writes indicator is a subtle count — escalate when stuck

If the Google Sheet is unreachable writes pile up silently. Once writeQueue.length > 5 or dbFails > 0, show a warning banner at the top: "Some scans haven't synced — keep scanning, we'll retry."

P3

Camera scanner button inside the AWB input is invisible

Scan-with-camera has to be triggered from the header icon. A second entry from inside the AWB field (📷 on the right edge of the input) would shave a click for the heavy-use path.

05 Cash Challan 6 issues

CashChallan.tsx — invoice creation, ledger, returns, WhatsApp reminders.

P0

Challan modal is 1056-line single component — high slip risk

Create / Edit / Return / Audit / Reminder / Ledger / Analytics / CashBook all mount through state flags in one component. Any bug in one mode blocks all others. Split into ChallanForm, ChallanReturnFlow, ChallanLedger, ChallanAnalytics — mount by route.

Fix: refactor by dialog (not state flag) — measurable P0 because testability is near-zero today
P1

Item rows in new challan don't support keyboard add

Each line has Description / Qty / Price / Discount. Tab order works until you hit "Total" — to add another row you must grab the mouse and click "+ Add item". Enter-in-last-field should append a row + focus the new SKU input.

P1

Return flow's source-challan search is disconnected from the list

Creating a return requires typing a challan number in a small search field. Given returns are usually done right after the user just saw the challan, add "Create return" as an action on each row in the main list — skips the search entirely.

P2

Status pills in the list are the only differentiator

A returned challan looks identical to a sale except for a grey "Return" badge and negative amount. Visually flip the row background to a faint red tint when is_return so operators don't double-count.

P2

Analytics modal opens as a separate pane — no comparison

Date picker gives you "this month" data but you can't compare against last month. A two-column mode (This period / Previous) or a %-change chip would make this actually useful.

P3

"ERP reminder" banner has no dismiss memory

Shows every session. Add a "Don't show again this week" with localStorage.

06 CashBook 4 issues

CashBook.tsx — expenses, cash-in sales, handovers with PIN confirmation.

P1

Handover PIN confirm has no lockout feedback

Wrong PIN shows confirmError text but doesn't slow subsequent attempts or count them. For a money-movement screen this is a security gap. Add exponential backoff + admin notification after 5 failed attempts.

P1

Opening balance is editable inline with no audit trail

Anyone with access can change the day's opening balance and there's no diff log shown back. Given the balance is the anchor of every day's reconciliation, changes need to appear in the handover breakdown with who/when.

P2

Tab switcher "Expenses / Sales / Handovers" is a small pill row

Visual weight doesn't match these being three fundamentally different datasets. Elevate to proper segmented control with counts per tab ("Expenses 14 · Sales 38 · Handovers 2").

P2

"Exclude paise" is a hidden toggle

Useful setting but buried. Move to a global display preference under the user menu so it applies everywhere money is shown.

07 Settings & Auth 5 issues

pages/Settings.tsx, pages/Login.tsx, components/settings/*.

P1

Login shows a generic error and loses typed email on failure

Auth error bubbles up as error.message inline. "Invalid login credentials" is fine, but the email field is preserved while the password is silently cleared — inconsistent. Also: no "Forgot password" link anywhere.

P1

Settings page gates the whole area behind "admin required"

Managers have legitimate needs (Locations, Brands) but see a dead-end "Admin access required" message. Split: Categories/Locations/Brands open to managers; Users/PackStation admin-only.

P2

Sign-out button has destructive colour at every render

Soft red by default in the sidebar — signals danger for a benign action. Use the neutral ghost style; reserve red for actual destructive confirmations.

P2

Settings tabs wrap on mobile with no current-tab indication above the fold

5 pill tabs wrap to 2 lines on narrow viewports; active pill may end up on row 2 and get missed. Horizontal-scroll row with sticky active indicator works better.

P3

Login card has 4 loop-in glow orbs that never settle

Intro animation is nice; the orbs stay at opacity. On older laptops this adds steady GPU work on the idle login screen. Fade to 0 after 4s.

08 Wins to keep 5 patterns

Patterns that are working — don't regress them during cleanup.

KEEP

Optimistic scan feedback in PackStation

Beep + flash + count-up fires before the DB write. This is exactly right for a high-throughput station — operator never waits on latency. The background queue with retries and beforeunload guard is thoughtful.

KEEP

Real-time subscriptions on list pages

Brand Tags, CashBook, Dashboard all subscribe to postgres_changes and apply row-level updates without a full refetch. Great for the multi-user shop-floor context.

KEEP

Lazy mount of heavy pages

App.tsx's mounted Set means BrandTagPrinter and CashChallan (the big ones) only mount on first visit. Saves significant first-paint time.

KEEP

Hash routing + popstate for in-app back

Mobile-friendly, browser-back opens drawer-to-dashboard correctly, PackStation history closes on back. Understated and working.

KEEP

Accent dot + underline treatment for section titles

The tiny gradient-dot-plus-divider before each screen title in the header is a clean, recognisable mark. Cheap to produce, strong visual identity. Use it more (currently only in Header).

Suggested fix sprint order. (1) Ship the global :focus-visible + type-scale bump + alert→toast swap — one-day change, touches every screen. (2) Break the Challan god-component into per-dialog files. (3) Non-blocking duplicate feedback in PackStation. (4) Clickable Dashboard alerts. (5) Mobile nav — add More tab. Everything else slots in after.
